# Vibe 作业提交平台上线前检查清单

本清单供课程负责人、服务端运维人员和助教使用。完成所有“阻塞项”后，再向学生发放安装包。

## 1. 发布学生客户端（阻塞项）

- [ ] 将当前 ZM 分支已验证的改动合并到正式发布分支。
- [ ] 将客户端版本从 `0.1.7` 升级为新版本，并创建新的不可变 Git 标签；已发给学生的标签绝不覆盖。
- [ ] 更新 Codex 插件与 Claude Code 安装包，使其指向该新标签。
- [ ] 安装包必须自动完成两件事：安装 `vibe-submit` 命令，以及安装用户级 `/submit-homework` Skill。
- [ ] 安装包必须预置正式 HTTPS 服务地址。学生注册时不得输入服务器 URL。
- [ ] 在一台从未安装过客户端的 Windows 电脑上验证安装、注册、提交和重试。

学生发布命令在 `v0.1.8` 标签创建并推送后生效：

```powershell
irm https://raw.githubusercontent.com/JasonLuo365/vibe-course-marketplace/v0.1.8/scripts/install-claude-submit.ps1 | iex
```

## 2. 部署生产服务端（阻塞项）

- [ ] 准备 Linux VPS、生产域名、DNS 解析，以及 80/443 入站访问。
- [ ] 安装 Docker Engine、Docker Compose 与 Caddy。
- [ ] 基于 `server/.env.example` 创建 `server/.env`；设置强随机 `VIBE_SESSION_SECRET`、LLM API Key、正式允许域名和生产环境开关。
- [ ] 保护 `server/.env`，禁止提交到 Git 或发送到聊天群。
- [ ] 使用 Docker Compose 构建并启动服务；应用 HTTP 仅绑定本机回环地址。
- [ ] 配置 Caddy 使用正式域名终止 HTTPS；从 VPS 外部访问 `https://<正式域名>/health` 并确认成功。
- [ ] 确认浏览器没有证书警告，学生客户端能严格验证 HTTPS 证书。

## 3. 数据与恢复（阻塞项）

- [ ] 确认 Docker 命名卷中保存 SQLite 数据库、上传 ZIP 和解压后的提交文件。
- [ ] 课前执行一次备份，并将至少一份备份复制到服务器外部。
- [ ] 在非生产环境演练一次恢复；恢复后执行健康检查和教师登录验证。
- [ ] 明确谁拥有恢复权限；恢复操作不能在课堂中临时执行。

## 4. 教学数据准备

- [ ] 创建教师账号，并将密码通过安全渠道交付。
- [ ] 创建课程，保存课程邀请码。
- [ ] 创建作业，核对开放时间、截止时间、上传大小限制和评分标准，保存作业编号。
- [ ] 仅向对应学生发送课程邀请码和作业编号。
- [ ] 预先确定重复提交规则：默认拒绝；学生明确确认覆盖后才提交新版本。

## 5. 端到端验收（阻塞项）

使用两个测试学生完成下列演练：

1. 安装正式发布版客户端和 Claude Code Skill。
2. 在终端执行 `vibe-submit setup`，确认不询问服务器 URL。
3. 在真实作业目录启动 Claude Code，并执行 `/submit-homework <作业编号>`。
4. 确认预览后上传；教师端应显示最新提交。
5. 修改作业后明确确认覆盖，验证教师端已切换到新 attempt。
6. 模拟网络失败，确认 `vibe-submit retry` 能重试原始上传包。
7. 验证教师端查看代码、报告、评估和学生端查看已发布反馈。

## 6. 课堂运行

- 以“提交已收到”为成功标准；AI 评估可以异步完成。
- 首次提交按学号或小组分批进行。
- 出现问题时先保存服务端日志和学生端错误信息；不要删除数据卷。
- 课堂期间不要切换版本、修改评分标准、重建数据库或覆盖已发布版本。

## 发布结论

只有当第 1、2、5 节全部完成，且备份已验证时，才向学生发布安装说明。
